# ce-site

**Curious Endeavor** website — clean rewrite.

Canonical repo for curiousendeavor.com. Replaces the legacy repo
`Assaf-Dagan/ce-website` which served the site out of a nested
`public/public/` directory alongside unrelated tooling, Discord
exports, PDFs, and Python scripts. This repo contains only the
website.

## Principles

1. **One publishing directory.** `site/` is Vercel's web root. No nesting.
2. **No cruft.** Tools, Discord exports, research, Python scripts stay
   outside this repo. Want to add something? If it's not served to users
   or used to build what's served, it goes in the workspace, not here.
3. **Components over duplication.** Shared UI lives in `site/components/`.
   No more 1,600-line HTML files copy-pasted between pitches.
4. **Prod depends on prod.** No more hero images loaded from
   `staging.curiousendeavor.com`. Anything a prod page references is
   committed to this repo.
5. **Small, reviewable commits.** One page per PR during migration.

## Getting started

```bash
# Install (only dev dependencies — no build step for the site)
npm install

# Regenerate the sitemap after adding/removing HTML pages
node scripts/build-sitemap.mjs

# Check every page for broken prod links (exits non-zero on failure)
node scripts/check-links.mjs --live

# Preview locally (any static file server works)
npx serve site
```

## Deployment

- **Prod branch:** `main` → `www.curiousendeavor.com`
- **Staging branch:** `staging` → `staging.curiousendeavor.com`
  (password-protected via Vercel)
- **Preview branches:** auto-deployed by Vercel on every push

Do not push directly to `main`. Merge `staging` → `main` when
`scripts/check-links.mjs --live` passes and QA is done.

## Where the old site lives

The legacy repo remains at `Assaf-Dagan/ce-website`, tag
`pre-migration-2026-04-09` on both `main` and `staging`. Use that as
a rollback point during the migration. After cutover (Phase 3), the
old repo is archived, not deleted.

See `project/migration-plan.md` for the phased rollout.
