### The `ce-hero` component — what it does and why it matters

Shared canonical hero for every page. One source of truth. Anything that includes it gets identical behavior — impossible to drift if the component is used.

**API via data attributes:**

| Attribute | Purpose | Example |
|---|---|---|
| `data-hero-title` | Headline HTML (supports `<em>` for red) | `"Your creative department, <em>minus the department</em>."` |
| `data-hero-subtitle` | Body paragraph under title | `"Strategy to execution..."` |
| `data-hero-label` | Small uppercase label | `"Pitch · April 2026"` |
| `data-hero-variant` | `"logo"` for figma-story style | (default = headline) |
| `data-hero-pool` | Which image pool JSON to load | `"default"` |
| `data-hero-pool-images` | Inline override (comma-sep) | `"a.jpg,b.jpg"` |
| `data-hero-height` | CSS for `--ce-hero-height` | `"70vh"` |

**Responsive behavior baked into `ce-hero.css`:**

- **≥901px (desktop):** 3×3 grid, 8% bleed, absolute-centered in 100vh `.ce-hero`
- **769–900px (tablet-ish):** same layout, bleed drops to 100% to fit viewport
- **≤768px (all mobile incl. landscape phones + portrait small tablets):** brick-wall 2-column with interlocking tall/short cells, fluid row heights `clamp(100px, 18vw, 160px)`, text cell spans both columns in middle. Ported from figma-story (Subject Matter Expert).
- **`prefers-reduced-motion: reduce`:** kills crossfade loop.

### The wordmark rule (LOCKED)

Documented in `project/decisions.md` and enforced by `scripts/check-hero-consistency.mjs`:

| Context | Treatment | Why |
|---|---|---|
| **Nav / top-left brand mark** | Canonical PNG `/assets/logos/ce-logo-red.png` | Small, pixel-perfect, no font-fallback flash |
| **Hero center text cell** | Live text "curious endeavor." rendered via Larken serif, red | Scales responsively, no aspect distortion |

Color is always red. Never black, white-on-dark, or tinted. No other wordmark asset.

### The guarantee layers (Layers 1–6)

This is the "how do we keep the hero identical everywhere" answer:

1. **Single source of truth:** `site/components/ce-hero/ce-hero.{css,js,pools/default.json}` — if two pages both `<link>`/`<script>` it, they're identical. Built ✓
2. **Namespaced CSS:** all rules `.ce-hero-*`, can't collide with page styles. Built ✓
3. **Variants via `data-*` attrs:** differences are declared in HTML, not hidden in overridden CSS. Built ✓
4. **CI lint (`check-hero-consistency.mjs`):** fails any PR that reinvents the hero, uses the wrong logo path, inlines a crossfade IIFE, or references staging assets. Built ✓, not yet wired into GitHub Actions
5. **Cutover retires old copies:** after DNS flip to new repo, physically impossible for old bespoke heroes to be served. Pending Phase 3
6. **Shared image pool:** `pools/default.json` is canonical — every page defaults to it. Built ✓
